# gotchu v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v1 `gotchu` Claude Code plugin — a Haiku-powered shoulder-tutor that watches the main agent end-of-turn and surfaces teaching moments via the conversation and a dynamic statusLine.

**Architecture:** Three Claude Code hooks. The two LLM-using hooks (PostToolUse, Stop) are `type:"agent"` with `model:"claude-haiku-4-5-20251001"` — they run inside the user's Claude Code session and bill via the existing subscription. The UserPromptSubmit hook is `type:"command"` (regex only, no LLM). The agent hooks use their built-in tool access (Bash/Read/Write) to mutate per-repo state files. A statusLine helper renders the pet line with TTL-aware transient messages.

**Tech Stack:** bash, jq, Claude Code native agent hooks, Claude Haiku 4.5 (`claude-haiku-4-5-20251001`).

**Implementation note — corrected from a planning misstep:** An earlier draft of this plan used `type:"command"` shell hooks calling the Anthropic API directly via curl + `ANTHROPIC_API_KEY`. That would have charged the user's API account separately from their Claude Code subscription. The corrected plan uses native `type:"agent"` hooks so all LLM work flows through the existing subscription.

**Latency note:** Agent hooks run synchronously (no `async` flag in the agent-hook schema). PostToolUse will add ~1-2s between tool calls while Haiku evaluates. We're accepting this as feature, not bug — for a "shoulder tutor" plugin, brief pauses give the user a moment to read what just happened. The mid-turn statusLine pings appear during those pauses.

**Runtime prerequisites for the user:**
- `bash`, `jq` on PATH (no `curl`, no API key — the plugin uses Claude Code's subscription)
- Claude Code installed (the plugin uses Claude-Code-only primitives)
- A repo where they've run `/gotchu init` to create the `.claude/gotchu/` state dir

```bash
cat > scripts/validate.sh <<'EOF'
#!/usr/bin/env bash
# gotchu self-test. Runs in a tempdir. Tests non-LLM components only;
# agent hooks (PostToolUse, Stop) are verified by manual smoke test.
set -euo pipefail

PLUGIN_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
TESTDIR="$(mktemp -d)"
PASS=0
FAIL=0

cleanup() { rm -rf "$TESTDIR"; }
trap cleanup EXIT

assert() {
  local desc="$1"; local actual="$2"; local expected="$3"
  if [ "$actual" = "$expected" ]; then
    echo "  ✓ $desc"
    PASS=$((PASS + 1))
  else
    echo "  ✗ $desc"
    echo "    expected: $expected"
    echo "    actual:   $actual"
    FAIL=$((FAIL + 1))
  fi
}

assert_file() {
  local desc="$1"; local path="$2"
  if [ -f "$path" ]; then
    echo "  ✓ $desc"
    PASS=$((PASS + 1))
  else
    echo "  ✗ $desc (file missing: $path)"
    FAIL=$((FAIL + 1))
  fi
}

assert_contains() {
  local desc="$1"; local haystack="$2"; local needle="$3"
  if echo "$haystack" | grep -qF "$needle"; then
    echo "  ✓ $desc"
    PASS=$((PASS + 1))
  else
    echo "  ✗ $desc (did not find: $needle)"
    FAIL=$((FAIL + 1))
  fi
}

echo "Running gotchu self-test in $TESTDIR"
cd "$TESTDIR"

assert "harness smoke test" "ok" "ok"

# === Component tests will be appended below as each task lands. ===

echo ""
echo "Results: $PASS passed, $FAIL failed"
[ "$FAIL" = "0" ]
EOF
chmod +x scripts/validate.sh
```

- [ ] **Step 3: Write hooks/on-prompt.sh**

```bash
cat > hooks/on-prompt.sh <<'EOF'
#!/usr/bin/env bash
# UserPromptSubmit hook. Regex-only: detect @gotchu directives in the user's prompt
# and update local state (.claude/gotchu/{state.json,intent.json}). No LLM.
set -euo pipefail

input=$(cat)
cwd=$(echo "$input" | jq -r '.cwd // .workspace.current_dir // "."' 2>/dev/null || echo ".")
prompt=$(echo "$input" | jq -r '.prompt // empty' 2>/dev/null || echo "")

STATE_DIR="$cwd/.claude/gotchu"
[ -d "$STATE_DIR" ] || exit 0

STATE_FILE="$STATE_DIR/state.json"
INTENT_FILE="$STATE_DIR/intent.json"
TMP="$STATE_FILE.tmp"

set_state() {
  jq "$1" "$STATE_FILE" > "$TMP" && mv "$TMP" "$STATE_FILE"
}

if [ -z "$prompt" ]; then
  exit 0
fi

if echo "$prompt" | grep -qE '@gotchu[[:space:]]+hush\b'; then
  set_state '. + {sticky:"hushed",emoji:"😴",text:"hushed",expires_at:0}'
elif echo "$prompt" | grep -qE '@gotchu[[:space:]]+wake\b'; then
  set_state 'del(.sticky) | .emoji="🐕" | .text="watching" | .expires_at=0'
elif echo "$prompt" | grep -qE '@gotchu[[:space:]]+more\b'; then
  jq -n '{command:"more"}' > "$INTENT_FILE"
elif echo "$prompt" | grep -qE '@gotchu[[:space:]]+what\b'; then
  jq -n '{command:"what"}' > "$INTENT_FILE"
else
  N=$(echo "$prompt" | grep -oE '@gotchu[[:space:]]+[1-9]\b' | grep -oE '[1-9]' | head -1 || true)
  if [ -n "$N" ]; then
    jq -n --arg n "$N" '{command:$n}' > "$INTENT_FILE"
  fi
fi

exit 0
EOF
chmod +x hooks/on-prompt.sh
```

# gotchu

A small Haiku-powered companion for Claude Code. Watches what the main agent is doing, surfaces teaching moments end-of-turn, reacts via the status line during turns. Runs on your existing Claude Code subscription via native agent hooks — no separate API billing.

## Activation

This plugin runs automatically via hooks once installed. There is no slash command to "invoke" it during a turn — the hooks fire on every PostToolUse and Stop event.

The one-time setup: `/gotchu init` creates `.claude/gotchu/` in the current repo so the hooks know they're in a gotchu-tracked workspace.

## Commands the user might type

The user may include `@gotchu ...` directives in their messages. These do NOT go to you (the main agent) — gotchu's UserPromptSubmit hook intercepts them and updates local state:

- `@gotchu more` — next debrief is in detailed mode
- `@gotchu N` (1–3) — detailed view of just lesson N
- `@gotchu what` — forces a debrief on the next Stop even if nothing notable accumulated
- `@gotchu hush` — mute pet for the rest of the session
- `@gotchu wake` — unmute

If the user types one of these directly to YOU, acknowledge briefly. Don't try to "be" the pet — the hook handles the action.

## How it works (so you can explain it if asked)

PostToolUse and Stop are `type:"agent"` hooks running Haiku 4.5. PostToolUse fires after each Bash/Write/Edit, with a ~1-2s pause while Haiku evaluates whether that tool call contained a teaching moment. If yes, it appends to `.claude/gotchu/findings.jsonl` and writes a transient statusLine state. Stop reads accumulated findings, composes a short debrief, emits it via `systemMessage`, and resets state. If nothing notable accumulated and no `@gotchu what` was issued, Stop stays silent.

A statusLine helper renders a live pet line from `.claude/gotchu/state.json`, with 40-second TTL on transient messages so the pet "reacts" mid-turn without sticking.

## Cost / dependencies

Uses your existing Claude Code subscription via native agent hooks. No separate API key. `bash` + `jq` need to be on PATH.

## Personality

Brief. Curious. Teaching-first. Never lectures; surfaces. Never apologizes for being quiet — silence is a feature, not a failure.
